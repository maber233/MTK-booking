# Email Configuration for EP-3 Booking System

## Current Issue Fix

The email system has been configured to work reliably in containerized environments like Railway. By default, emails are now saved to files instead of being sent via sendmail.

## Configuration Options

The system supports three email transport methods:

### 1. File-based (Default - Recommended for Development/Testing)
Emails are saved as files in the `data/mails/` directory instead of being sent.

**Railway Environment Variables:**
```bash
MAIL_TYPE=file
MAIL_FROM=info@yourdomain.com
```

### 2. SMTP (Recommended for Production)
Send emails via SMTP server (like Gmail, SendGrid, Mailgun, etc.)

**Railway Environment Variables:**
```bash
MAIL_TYPE=smtp
MAIL_FROM=info@yourdomain.com
MAIL_HOST=smtp.gmail.com
MAIL_USER=your-email@gmail.com
MAIL_PASSWORD=your-app-password
MAIL_PORT=587
MAIL_AUTH=plain
```

### 3. SMTP with TLS (Secure SMTP)
Same as SMTP but with TLS encryption (recommended for production).

**Railway Environment Variables:**
```bash
MAIL_TYPE=smtp-tls
MAIL_FROM=info@yourdomain.com
MAIL_HOST=smtp.gmail.com
MAIL_USER=your-email@gmail.com
MAIL_PASSWORD=your-app-password
MAIL_PORT=587
MAIL_AUTH=plain
```

## Setting Up SMTP with Popular Providers

### Gmail
1. Enable 2-factor authentication on your Google account
2. Generate an App Password: https://myaccount.google.com/apppasswords
3. Use these settings:
   ```bash
   MAIL_HOST=smtp.gmail.com
   MAIL_PORT=587
   MAIL_AUTH=plain
   ```

### SendGrid
1. Create a SendGrid account and get an API key
2. Use these settings:
   ```bash
   MAIL_HOST=smtp.sendgrid.net
   MAIL_PORT=587
   MAIL_USER=apikey
   MAIL_PASSWORD=your-sendgrid-api-key
   MAIL_AUTH=plain
   ```

### Mailgun
1. Create a Mailgun account and get SMTP credentials
2. Use these settings:
   ```bash
   MAIL_HOST=smtp.mailgun.org
   MAIL_PORT=587
   MAIL_USER=your-mailgun-username
   MAIL_PASSWORD=your-mailgun-password
   MAIL_AUTH=plain
   ```

## How to Configure in Railway

1. Go to your Railway project dashboard
2. Click on the "Variables" tab
3. Add the environment variables listed above for your chosen email method
4. Redeploy your application

## Testing Email Configuration

### With File Transport (Current Default)
- Emails will be saved in the `data/mails/` directory
- You can check if emails are being generated by looking at the files in this directory
- This is perfect for testing that the email functionality works without actually sending emails

### With SMTP Transport
- Emails will be sent to the actual recipients
- Check your email provider's logs for delivery status
- Make sure your domain/IP is not blacklisted

## Troubleshooting

1. **Emails not being generated**: Check application logs for PHP errors
2. **SMTP connection issues**: Verify host, port, username, and password
3. **Authentication failures**: Try changing `MAIL_AUTH` from `plain` to `login`
4. **TLS/SSL issues**: Make sure your SMTP provider supports the encryption method you're using

## How User Activation Works

1. **User registers** → Account created in "disabled" state
2. **System sends activation email** → Contains activation link  
3. **User clicks link** → Account becomes "enabled"
4. **User can now login** → Full access to booking system
